## 10. 플레이스홀더 이미지 운용 방침

에셋 원본이 준비되기 전까지, **플레이스홀더 이미지로 먼저 퍼블리싱을 진행**합니다.

### 10-1. 플레이스홀더 규칙

| 에셋 유형 | 플레이스홀더 방식 | 예시 |
|----------|----------------|------|
| 배경 사진 (히어로, 배너 등) | 단색 배경 + 에셋명 텍스트 오버레이 | `background: #888; /* hero-bg.jpg */` |
| 메뉴/후기 사진 | `placehold.co` 서비스 또는 CSS 단색 박스 | `<img src="https://placehold.co/290x218/ECE5D8/505050?text=대도카세">` |
| 로고 (SVG) | 텍스트로 대체 (갈매기대도) + 주석 | `<!-- TODO: logo-horizontal-white.svg -->` |
| 파비콘 | CSS로 원형 + 이니셜 표시 | `background: #EC6937; border-radius: 50%;` |
| 아이콘 (SVG) | CSS로 도형 표시 또는 유니코드 문자 | `content: "☎";` 또는 빈 원형 |
| 패턴 타일 | CSS 반복 패턴으로 대체 | `repeating-linear-gradient(...)` |
| 아치형 텍스트 | 일반 텍스트 + 주석 | `<p>갈매기살의 새로운 기준</p> <!-- arch-text.svg -->` |
| 폰 목업 | CSS border로 폰 형태 표현 | `border-radius: 40px; border: 3px solid #333;` |

### 10-2. 교체 편의를 위한 코드 규칙

1. **모든 플레이스홀더에 `TODO` 주석 표기**
   ```html
   <!-- TODO: 실제 에셋으로 교체 → hero-bg.jpg -->
   <div class="hero" style="background: #555;"></div>
   ```

2. **이미지 경로는 최종 파일명으로 미리 설정** (파일만 넣으면 바로 적용되도록)
   ```html
   <!-- 에셋 준비 시 assets/images/hero-bg.jpg 파일만 추가하면 자동 적용 -->
   <img src="assets/images/menu-dadocase.jpg"
        alt="대도카세"
        onerror="this.style.background='#ECE5D8'; this.alt='대도카세 (이미지 준비중)'">
   ```

3. **CSS 배경 이미지도 최종 경로로 선언하되, fallback 색상 지정**
   ```css
   .hero {
     background: #555 url('assets/images/hero-bg.jpg') center/cover no-repeat;
     /* fallback: 이미지 없으면 #555 배경 표시 */
   }
   ```

4. **SVG 아이콘은 `assets/icons/` 경로로 미리 참조**
   ```html
   <img src="assets/icons/step-01-inquiry.svg" alt="가맹 문의 접수"
        onerror="this.style.display='none'">
   ```

### 10-3. 에셋 교체 절차

1. Figma에서 `갈매기대도_에셋_Export_가이드.md`의 노드 ID 참고하여 Export
2. 가이드에 명시된 파일명으로 저장
3. `assets/` 하위 폴더에 배치:
   - `assets/images/` — JPG/WebP 사진
   - `assets/svg/` — 로고, 벡터, 패턴
   - `assets/icons/` — 아이콘 SVG
4. 브라우저 새로고침 → 자동 적용 확인
5. `TODO` 주석 검색하여 잔여 플레이스홀더 정리

### 12-1. 키카피 텍스트 — 섹션별 실제 적용 텍스트
