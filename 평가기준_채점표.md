## 평가 기준 (총 100점)

### 1. 시안 일치도 (40점)

| No | 세부 항목 | 배점 | 평가 기준 |
|----|-----------|------|-----------|
| 1-1 | 레이아웃 구조 | 10 | 섹션 배치 순서, 좌우/상하 간격, 정렬이 Figma 시안과 일치하는가 |
| 1-2 | 컬러 정확도 | 8 | 디자인 토큰(MainGreen #006343, MainOrange #EC6937 등) 적용이 정확한가 |
| 1-3 | 타이포그래피 | 8 | 폰트(Pretendard), 크기, 굵기, 자간(letter-spacing), 행간(line-height) 일치 여부 |
| 1-4 | 컴포넌트 완성도 | 8 | GNB, 카드, 테이블, 폼, 배지, 탭 등 개별 UI 컴포넌트 구현 수준 |
| 1-5 | 이미지/에셋 배치 | 6 | 플레이스홀더 포함, 위치/크기/비율/border-radius 정확도 |

**채점 가이드:**
- 10/10: Figma 시안과 픽셀 수준 일치
- 7~9: 전체적으로 일치하나 미세한 차이 존재
- 4~6: 구조는 맞으나 세부 수치에서 차이가 큼
- 1~3: 시안과 상당한 차이
- 0: 해당 항목 미구현

### 2. 코드 품질 (25점)

| No | 세부 항목 | 배점 | 평가 기준 |
|----|-----------|------|-----------|
| 2-1 | 시맨틱 HTML | 7 | header, nav, main, section, article, footer 등 의미있는 태그 사용 |
| 2-2 | CSS 구조 | 7 | CSS 변수 활용, 중복 최소화, 일관된 네이밍 컨벤션 (BEM 등) |
| 2-3 | 코드 가독성 | 6 | 들여쓰기, 주석, 파일 분리, 코드 정돈 수준 |
| 2-4 | 유지보수성 | 5 | 에셋 교체 용이성, 텍스트 수정 편의성, TODO 주석 표기 |

**채점 가이드:**
- 만점: 실무 수준의 깔끔한 코드, 재사용성 높음
- 70%: 동작하지만 일부 개선 필요
- 50%: 기본적으로 동작하나 구조적 문제 있음
- 30% 이하: 유지보수 어려움

### 3. 반응형 대응 (20점)

| No | 세부 항목 | 배점 | 평가 기준 |
|----|-----------|------|-----------|
| 3-1 | 모바일 레이아웃 | 8 | 375px 기준 모바일 시안(33:791)과의 일치도 |
| 3-2 | 브레이크포인트 처리 | 6 | PC↔Mobile 전환의 자연스러움, 중간 해상도 대응 |
| 3-3 | 타이포 스케일 조정 | 6 | 모바일 폰트 크기 축소 정확도 (52→32, 42→21 등) |

**채점 가이드:**
- 만점: PC/Mobile 모두 시안과 일치, 중간 해상도에서도 깨지지 않음
- 70%: 대부분 대응되나 일부 섹션에서 이슈
- 50%: 기본 반응형은 적용되나 시안과 차이가 큼
- 30% 이하: 반응형 미적용 또는 심각한 레이아웃 깨짐

| No | 세부 항목 | 배점 | 평가 기준 |

| 4-2 | 모바일 햄버거 메뉴 | 4 | 햄버거 아이콘 클릭 → 풀스크린 메뉴 열기/닫기 |
| 4-3 | 탭 전환 | 4 | 상차림 섹션 구이류/곁들임 탭 전환 동작 |
| 4-4 | 기타 인터랙션 | 3 | 슬라이드 좌우 이동, 호버 효과, 폼 기본 동작 등 |

**채점 가이드:**
- 만점: 모든 인터랙션 정상 동작
- 3/4: 핵심 기능 동작, 세부 효과 일부 미흡
- 2/4: 기본 동작만 구현
- 1/4 이하: 인터랙션 대부분 미구현
