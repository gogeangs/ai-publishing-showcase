## 1장. 왜 이 테스트를 했는가

### 배경

퍼블리싱(디자인 시안 → HTML/CSS 코드 변환) 작업은 반복적이고 시간이 많이 걸립니다.
AI가 이 작업을 대신하거나 보조할 수 있다면, 제작 기간과 비용을 줄일 수 있습니다.

### 질문

> "AI에게 어떻게 지시해야 완성본에 가장 가깝게 만들 수 있는가?"

### 테스트 설계

같은 Figma 시안(갈매기대도 랜딩페이지)을 **4가지 다른 방법**으로 AI에게 지시하고,
결과물을 **실제 배포된 완성본**과 비교했습니다.

| 테스트 | 방법 요약 | 비유 |
|--------|----------|------|
| test-1 | 상세한 업무요청서(.md) + Figma 참조 | "사수가 신입에게 상세 스펙 넘기며 시킨 것" |
| test-2 | Figma URL만 전달 | "디자인 파일 링크 하나 던지고 알아서 해봐" |
| test-3 | Figma 로컬 MCP (이미지 자동 추출) | "Figma 파일 열어놓고 필요한 거 직접 꺼내쓰며 만들어봐" |
| test-4 | 업무요청서 v2 + Figma 로컬 MCP | "핵심 스펙만 정리해주고, Figma도 직접 보면서 만들어봐" |

### 비교 기준: 완성본(레퍼런스)

레퍼런스는 **사람이 직접 만든 실 배포 사이트**입니다.
이 완성본과 AI 결과물을 **브라우저에서 사용자 눈으로 비교**했습니다.

완성본의 주요 특징:
- 12개 섹션으로 구성된 단일 페이지 랜딩
- 실사 사진 30개 이상 (음식, 매장, 로고 등)
- 비디오 배경, Swiper 슬라이더, GSAP 스크롤 애니메이션
- 10단계 반응형 (1280~370px)
- jQuery, Swiper.js, GSAP, AOS 등 라이브러리 활용

## 3장. 결과 요약

### 3-1. 시각적 유사도 (사용자가 브라우저에서 보는 화면 기준)

| 순위 | 테스트 | 시각적 유사도 | 한 줄 평가 |
|:----:|--------|:-----------:|----------|
| **1위** | test-4 (업무요청서 v2 + MCP) | **약 70~75%** | 실제 이미지 + 슬라이더/스크롤 애니메이션까지 동작 |
| 2위 | test-3 (Figma 로컬 MCP) | 약 65% | 실제 이미지가 표시되어 "진짜 웹사이트"처럼 보임 |
| 3위 | test-1 (업무요청서) | 약 35% | 구조는 완벽하지만 이미지가 전부 회색 |
| 4위 | test-2 (URL만) | 약 30% | 이미지도 없고 섹션도 일부 누락 |

### 3-2. 왜 이런 결과가 나왔는가

**한 마디로: 이미지 + 인터랙션 때문입니다.**

완성본 화면의 약 60%는 이미지(사진, 로고, 비디오)가 차지합니다.
test-3과 test-4는 Figma에서 이미지를 자동으로 추출했고,
test-1과 test-2는 이미지가 하나도 없어서 회색 플레이스홀더만 보입니다.

test-4가 test-3보다 5~10%p 높은 이유:
- **12/12 섹션 전부 구현** (test-3은 11/12, 배경이미지 섹션 누락)
- **슬라이더가 실제로 동작** (CSS transform 기반 커스텀 슬라이더)
- **스크롤 진입 애니메이션** (IntersectionObserver로 fadeUp 효과)
- **폼 유효성 검증 완전 구현** (이름/전화/동의 체크 + alert)

| | test-1 | test-2 | test-3 | test-4 |
|---|:---:|:---:|:---:|:---:|
| 자동 추출된 이미지 | **0개** | **0개** | **32개** | **30개** |
| 동작하는 인터랙션 | 3개 | 5개 | 5개 | **11개** |
| 사용자가 보는 느낌 | 와이어프레임 | 와이어프레임 | 실제 웹사이트 | **가장 완성된 웹사이트** |

### 3-3. 코드 구조 품질 (100점 기준, 코드 관점)

시각적 유사도와 별개로, 코드 자체의 품질도 평가했습니다:

| | test-1 | test-2 | test-3 | test-4 | 완성본(사람) |
|---|:---:|:---:|:---:|:---:|:---:|
| 시안 일치도 (40점) | **36** | 33 | 34 | **37** | 39 |
| 코드 품질 (25점) | **24** | **24** | **24** | **24** | 11 |
| 반응형 (20점) | **18** | 17 | 17 | **18** | **20** |
| 기능 구현 (15점) | 13 | 11 | 13 | **14** | **15** |
| **합계** | 91 | 85 | 88 | **93** | 85 |

주목할 점:
- **test-4가 코드 품질에서도 1위** (93점) — 시각적 유사도와 코드 품질 모두 1위인 유일한 테스트
- **코드 품질은 AI가 사람보다 높음** (AI 24/25 vs 사람 11/25)
- AI 코드: CSS 변수, BEM 네이밍, 시맨틱 HTML, 섹션 주석이 일관적
- 사람 코드: div 기반 마크업, CSS 변수 미사용, 시맨틱 태그 부재
- **반응형은 사람이 앞섬** (사람 10단계 vs AI 768px 하나만)
- **기능 구현: test-4가 가장 근접** (14/15점 — 슬라이더+애니메이션+폼 검증)

### 3-4. 시각적 유사도 vs 코드 품질 — test-4는 둘 다 1위

| 평가 기준 | 1위 | 2위 | 3위 | 4위 |
|----------|-----|-----|-----|-----|
| **시각적 유사도** (사용자 눈) | **test-4** (70~75%) | test-3 (65%) | test-1 (35%) | test-2 (30%) |
| **코드 품질** (개발자 눈) | **test-4** (93점) | test-1 (91점) | test-3 (88점) | test-2 (85점) |

이전 3개 테스트에서는 시각적 유사도 1위(test-3)와 코드 품질 1위(test-1)가 달랐지만,
**test-4는 업무요청서 v2 + MCP 조합으로 두 측면 모두에서 1위**를 달성했습니다.

- 이후 유사 프로젝트에 스펙 문서를 재활용할 수 있는 경우

### test-2: Figma URL만 제공

**강점**
- 준비 시간 0분 — URL 하나만 복사하면 됨
- 5~10분 만에 전체 구조 확인 가능

**약점**

- 1~2개 섹션 누락 가능

- 여러 시안 중 어떤 것을 개발할지 비교할 때
- Figma Pro 플랜이 없을 때

### test-3: Figma 로컬 MCP

**강점**
- **이미지 32개를 Figma에서 자동 추출** (가장 큰 차별점)

- 브라우저에서 열면 "실제 웹사이트"처럼 보임
- 준비 시간이 짧음 (MCP 설정 20분, 최초 1회)

**약점**
- Figma Professional 플랜 이상 필요 (유료)
- 배경이미지 섹션 누락 (1개)
- 추출된 이미지 용량이 큼 (최적화 필요)
- 슬라이더/스크롤 애니메이션 미구현

**적합한 상황**
- 빠른 결과가 필요하고 업무요청서 작성할 시간이 없을 때
- 이미지가 많은 랜딩페이지/프로모션 페이지
- 디자인 확정 후 빠른 1차 결과물이 필요한 경우

### test-4: 업무요청서 v2 + Figma 로컬 MCP (권장)

**강점**
- **12/12 섹션 전부 구현** (업무요청서의 섹션 목록이 누락 방지)
- **이미지 30개 자동 추출** (MCP 활용)
- **슬라이더, 스크롤 애니메이션, 폼 검증이 모두 동작** (유일)
- 업무요청서 v2는 327줄로 간결 (v1의 893줄 대비 63% 감소)
- 업무요청서 작성을 Claude가 자동화 (PM은 검수만, 25~40분)
- 시각적 유사도와 코드 품질 모두 1위

**약점**
- Figma Professional 플랜 이상 필요 (유료)
- 업무요청서 v2 준비에 25~40분 소요 (test-2, test-3 대비)
- Swiper가 아닌 커스텀 슬라이더 (Swiper 교체 권장)
- 추출된 이미지 용량 최적화 필요

**적합한 상황**
- **실무 제작 (가장 빠르게 완성본에 도달)** — 최우선 권장
- 섹션이 10개 이상인 복잡한 페이지
- 디자인 확정 후 빠른 개발이 필요한 경우
- 인터랙션이 중요한 랜딩페이지

## 8장. 테스트 과정에서 발견된 문제점

### 8-1. 디자인 시안 측 문제

테스트 과정에서 **Figma 시안의 준비 상태**가 결과에 큰 영향을 미쳤습니다:

| 문제 | 영향 | 원인 |
|------|------|------|
| **키카피 텍스트 오버라이드 미적용** | 모든 섹션 제목이 "100% 고객 만족 후기"로 동일 | 컴포넌트 인스턴스의 텍스트 수정이 안 됨 |
| **로고가 마스크 레이어** | AI가 로고를 추출 못하고 텍스트로 대체 | 로고가 Export Ready 상태가 아님 |
| **인터랙션 동작 미명시** | AI가 정적 UI만 생성, 동작 로직 누락 | 슬라이더 방향, 탭 전환 방식 등 미확인 5건 |
| **업무요청서와 시안 불일치** | AI가 어느 쪽을 따를지 혼란 | 푸터 배경색 등 |
| **특수 폰트 미처리** | 아치형 텍스트가 일반 글씨로 렌더링 | 웹폰트 없는 서체를 이미지 처리 안 함 |
| **장식 섹션 구분 불명확** | AI가 배경이미지 섹션을 누락 (2/4 테스트) | 섹션 이름/구조가 모호 |

### 8-2. AI 도구 측 한계

| 한계 | 설명 | test-4에서의 개선 |
|------|------|-----------------|
| 슬라이더/애니메이션 미구현 | Swiper, GSAP, AOS 등 외부 라이브러리 연동 불가 | **커스텀 JS로 대체 구현** |
| 비디오 배경 미적용 | 정적 이미지로만 처리 | 미개선 |
| 단일 브레이크포인트 | 768px 하나만 — 태블릿 등 중간 해상도 미대응 | 미개선 |
| 이미지 용량 미최적화 | MCP 추출 시 원본 해상도 그대로 (5MB 이미지 발생) | 미개선 |
| Pretendard 외 폰트 미적용 | 추가 폰트(GmarketSans 등) 자동 적용 안 함 | 미개선 |

### 8-3. 업무요청서 v2의 효과 (test-4에서 확인)

업무요청서 v2가 해결한 문제들:

| 기존 문제 | v2에서의 해결 |
|----------|-------------|
| 섹션 누락 (2/3 테스트) | 섹션 목록 명시 → **12/12 완전 구현** |
| 인터랙션 미구현 (3/3 테스트) | 인터랙션 요구사항 명시 → **슬라이더·애니메이션·폼 검증 구현** |
| 텍스트 오버라이드 미적용 | 실제 텍스트를 업무요청서에 기재 → **정확한 텍스트 반영** |
| 시안-문서 충돌 | MCP가 읽을 수 있는 정보는 문서에 미기재 → **충돌 제거** |

### 8-4. 결론

> **AI 결과물의 품질은 "시안의 준비 상태"와 "프롬프트 방법"에 좌우됩니다.**
> 시안이 잘 정리되어 있고, 업무요청서 v2 + MCP를 사용하면 **70~75%까지 자동화**됩니다.
> 나머지 25~30%(Swiper 교체, 세밀한 반응형, 비디오 배경, QA)는 사람이 해야 합니다.

## 12장. 최종 결론

### 질문에 대한 답

> "AI에게 어떻게 지시해야 완성본에 가장 가깝게 만들 수 있는가?"

**답: 업무요청서 v2 + Figma Desktop Local MCP (test-4 방식, 워크플로우 C)를 사용하세요.**

가장 큰 이유 3가지:
1. **이미지를 자동으로 추출** — 화면의 60%를 차지하는 이미지가 있어야 "완성된 것처럼" 보임
2. **섹션 누락을 방지** — 업무요청서에 섹션 목록을 명시하면 12/12 전부 구현
3. **기본 인터랙션까지 자동 구현** — 슬라이더, 스크롤 애니메이션, 폼 검증이 동작

### 4가지 핵심 메시지

**1. AI 퍼블리싱은 실무에 도입 가능합니다.**
- 3~5일 → **약 1일** (65~75% 단축)
- 코드 품질은 사람보다 일관적 (AI 24/25 vs 사람 11/25)
- Figma MCP로 이미지 수급 + 기본 인터랙션까지 자동화

**2. test-4 (워크플로우 C)가 가장 효율적입니다.**
- 시각적 유사도 1위 (70~75%) + 코드 품질 1위 (93점)
- 업무요청서 v2 작성은 25~40분으로 간결 (Claude가 자동 작성)
- 사람 보강 작업이 가장 적음 (5~9시간)

**3. 단, AI만으로는 100% 완성이 안 됩니다.**
- Swiper 교체, 세밀한 반응형 (태블릿 등), 비디오 배경은 사람이 추가해야 함
- AI가 가끔 섹션을 누락하므로 반드시 검수 필요 (업무요청서 v2로 대폭 개선됨)

**4. 결과 품질은 디자이너의 시안 준비 상태에 달려 있습니다.**
- 텍스트 오버라이드 미적용 → AI가 잘못된 텍스트 사용
- 에셋 Export 미설정 → 이미지 추출 실패
- 인터랙션 미명시 → AI가 정적 UI만 생성
- **"시안의 완성도 = AI 결과물의 완성도"**
