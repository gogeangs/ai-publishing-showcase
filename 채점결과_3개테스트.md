## 개별 분석

### test-1 (.md 업무요청서 기반) — 시각적 유사도 약 35%

**강점: 구조 완성도 1위**
- 12개 섹션 전부 빠짐없이 구현 (유일)
- "배경이미지" 섹션 (블러 + 누끼 + "0.3% 희소성") 구현 (유일)
- 25개 TODO 주석으로 에셋 교체 가이드 제공
- 타이포 스케일 PC/Mobile 가장 정확 (52→32, 42→21)

**약점: 이미지 전무**
- 모든 이미지가 placehold.co 회색 박스
- 로고도 텍스트 대체
- 비디오 배경 없음
- 슬라이더/애니메이션 미동작

**레퍼런스 대비 시각적 차이**
- 이미지가 차지하는 면적(약 60%)이 전부 회색 → 레퍼런스와 전혀 다른 인상
- 구조(섹션 순서, 카드 배치, 테이블)는 정확하지만, 사용자 눈에는 "미완성"으로 보임

### test-2 (Figma URL만 제공) — 시각적 유사도 약 30%

**강점: 최소 입력으로 기본 구조 확보**
- Figma URL 하나만으로 전체 구조 파악
- 창업비용 테이블이 레퍼런스와 거의 동일
- CSS 자동스크롤 리뷰 애니메이션 (독자적 구현)
- ARIA 접근성이 4개 중 가장 꼼꼼

**약점: 구조 누락 + 이미지 전무**
- "배경이미지" 섹션 완전 미구현
- 모든 이미지가 div 플레이스홀더
- 탭 전환 시 실제 컨텐츠 교체 미구현
- 모바일 타이포 스케일 편차 큼

**레퍼런스 대비 시각적 차이**
- test-1과 마찬가지로 이미지 전무 → 와이어프레임 느낌
- 배경이미지 섹션 자체가 없으므로 스크롤 시 "무언가 빠진 느낌"

---

### test-3 (Figma Desktop Local MCP) — 시각적 유사도 약 65%

**강점: 실제 이미지로 레퍼런스와 유사**
- **Figma에서 에셋 32개 자동 추출** (PNG 이미지 + SVG 로고/아이콘)
- 히어로에 실제 매장 사진, 브랜드 섹션에 실사 이미지
- SVG 로고로 GNB/히어로가 레퍼런스와 거의 동일
- 붓터치 SVG, 말풍선 꼬리, 패턴 등 디테일 에셋 포함

**약점: 섹션 누락 + 인터랙션 부재**
- "배경이미지" 섹션 (블러 + 누끼 + 희소성 프레임) 완전 누락
- Swiper 슬라이더 미동작
- GSAP/AOS 스크롤 애니메이션 없음
- 모바일 GNB 높이 64px (레퍼런스는 50px)

**레퍼런스 대비 시각적 차이**
- 이미지가 있으므로 전체적인 "느낌"이 레퍼런스와 유사
- 정적 페이지이므로 스크롤 시 애니메이션/슬라이드 효과는 없음
- 배경이미지 섹션 누락으로 중간에 한 섹션이 비어있음

---

### test-4 (업무요청서 v2 + Figma Desktop Local MCP) — 시각적 유사도 약 70~75%

**강점: 구조 + 에셋 + 인터랙션 3박자**
- **12/12 섹션 전부 구현** (업무요청서 v2의 섹션 목록이 누락 방지)
- **Figma에서 에셋 30개 자동 추출** (이미지 12 + SVG 11 + 아이콘 7)
- **커스텀 슬라이더 동작** (CSS transform 기반 좌우 이동 + 화살표 네비게이션)
- **탭 콘텐츠 실제 교체** (display flex/none 토글, 단순 클래스 토글이 아님)
- **IntersectionObserver 스크롤 진입 효과** (fadeUp 애니메이션, 라이브러리 없이 직접 구현)
- **폼 유효성 검증 완전 구현** (이름/전화/동의 체크 + alert 피드백)
- **하단 문의 바 2개** (히어로 하단 + 푸터 하단) 각각 독립 유효성 검증
- 글래스모피즘 (backdrop-filter: blur(4px)) 차별점 카드에 적용
- GNB 투명→흰색 스크롤 전환 + 로고 스왑 (dark↔white)

**약점: 디테일 차이**
- 배경이미지 섹션은 구현되었으나 블러+누끼 레이어 효과가 레퍼런스보다 단순
- 탭 활성 그라데이션이 linear (레퍼런스는 radial)
- 비용 테이블 헤더가 녹색 (레퍼런스는 베이지)
- 중간배너에 id 속성 누락
- 고객후기 자동 슬라이드 미구현 (정적 표시)
- 곁들임 탭 콘텐츠가 플레이스홀더 ("곁들임 메뉴 1~4")

**레퍼런스 대비 시각적 차이**
- 에셋 + 인터랙션 조합으로 4개 테스트 중 레퍼런스와 가장 유사
- 스크롤 시 섹션이 서서히 나타나는 애니메이션 효과가 있어 "동적인 느낌"
- 슬라이더 좌우 이동이 동작하여 상차림 섹션이 레퍼런스에 가까움
- Swiper/GSAP 같은 외부 라이브러리 없이 순수 JS로 구현한 점이 특이

## 결론

1. **시각적 유사도는 이미지 에셋이 결정한다** — 코드 구조가 아무리 좋아도 이미지가 없으면 와이어프레임
2. **업무요청서 v2 + MCP 조합(test-4)이 가장 효율적** — 에셋 자동 추출 + 섹션 누락 방지 + 인터랙션 기본 구현
3. **test-4는 test-3 대비 5~10%p 향상** — 12/12 섹션 완성 + 슬라이더/스크롤 애니메이션/폼 검증이 추가된 결과
4. **상세 스펙 문서의 진화**: test-1의 893줄 수동 작성(2~4시간) → test-4의 327줄 자동 작성(25~40분)으로 효율 개선
5. **인터랙션 자동 구현이 핵심 차별점** — test-1~3은 슬라이더·스크롤 애니메이션·폼 검증 중 최대 1개만 구현했으나, test-4는 3가지 모두 구현
6. **최적 워크플로우**: 업무요청서 v2로 구조 완성도 확보 + Figma MCP로 에셋 추출 → 사람은 Swiper 교체 + 반응형 보강에만 집중
