# AI 퍼블리싱 디자이너 가이드라인 — Figma 파일 구성안

이 문서는 디자이너가 Figma에서 가이드라인 파일을 제작할 때 사용하는 **페이지 구성, 프레임 레이아웃, 텍스트 블록**을 정리한 설계도입니다.
복사해서 바로 붙여넣을 수 있는 텍스트를 포함하고 있으며, 각 페이지의 프레임 구조와 디자인 참고사항을 안내합니다.

```

**레이어 구조 비교 다이어그램**

Bad 예시 (테두리 2px #D9263C, 배경 #FFF5F5):
```
📁 가로 로고 (흰색)
  📁 마스크 그룹
    🔲 직사각형 (클리핑 영역)
    📝 로고 벡터 패스 1
    📝 로고 벡터 패스 2
    📝 로고 벡터 패스 3
```

Good 예시 (테두리 2px #006343, 배경 #F0FFF4):
```
📁 가로 로고 (흰색)  [Export: SVG]
  🔲 Flatten된 단일 벡터 레이어
```

**에셋별 권장 형식 테이블**

| 에셋 유형 | 권장 형식 | 이유 |
|-----------|----------|------|
| 로고, 아이콘 | SVG | 벡터라서 해상도 독립적, 파일 작음 |
| 사진 (음식, 매장) | JPG/WebP | 용량 최적화 |
| 투명 배경 이미지 (누끼) | PNG | 투명도 필요 |
| 특수 폰트 텍스트 | SVG (통이미지) | 웹폰트 없는 서체는 이미지 대체 |
| 패턴 타일 | PNG | CSS background-repeat로 사용 |
| 복합 구성 (텍스트+장식) | SVG (통이미지) | 레이어 분리하면 재현 불가 |

**체크리스트**
```
□ Export될 에셋은 Flatten(병합) 처리
□ 또는 Export 설정(SVG/PNG)을 미리 지정
□ 벡터 에셋(로고, 아이콘) → SVG Export 설정
□ 사진 에셋(음식, 매장) → PNG 또는 JPG Export 설정
□ 복잡한 구성(아치형 텍스트, 붓터치+텍스트) → 통이미지 SVG Export 설정
```

**매핑 예시 다이어그램**

| Figma Variable | → | CSS Variable |
|----------------|---|-------------|

| MainOrange: #EC6937 | → | --main-orange: #EC6937 |
| 본문컬러: #505050 | → | --body-color: #505050 |
| Black: #111111 | → | --black: #111111 |

**Variable 네이밍 규칙**

권장 (초록 배경 #F0FFF4):
```
color/main-green      → --color-main-green
color/main-orange     → --color-main-orange
color/body            → --color-body
bg/beige              → --bg-beige
text/heading-1        → --text-heading-1
```

비권장 (빨간 배경 #FFF5F5):
```
본문컬러               → --본문컬러 (한글 변수명)
색상1                  → --색상1 (의미 파악 불가)
#006343 직접 입력       → (Variable 미사용)
```

**체크리스트**
```
□ 브랜드 색상을 Figma Variables로 정의
□ 모든 색상을 직접 입력(#006343) 하지 말고 Variable로 적용
□ 폰트 크기도 Variables로 정의 (권장)
□ 간격(spacing)도 Variables로 관리 (권장)

**미확인 항목 테이블**

| 미확인 항목 | AI의 대응 |
|------------|----------|

### 페이지 제목

**제목** (Pretendard Bold / 36px / #111111)
```
규칙 우선순위 맵
```

**설명** (Pretendard Regular / 16px / #505050)
```
10개 규칙을 영향도(Y축)와 난이도(X축) 기준으로 배치했습니다.
좌측 상단에 가까울수록 먼저 적용해야 합니다.
```

### 2x2 매트릭스 다이어그램

- X축 라벨: "난이도" (왼쪽: 쉬움, 오른쪽: 어려움)
- Y축 라벨: "영향도" (아래: 낮음, 위: 높음)
- 각 규칙을 카드(라운드 8px)로 배치
- 카드 색상은 영역에 따라 구분
