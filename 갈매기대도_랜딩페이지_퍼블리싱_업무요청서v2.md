## 5. 에셋 규칙

### 5-1. 폴더 구조

```
test-c/
├── index.html
├── style.css
├── script.js
└── assets/
    ├── images/     ← JPG/WebP/PNG 사진
    ├── svg/        ← 로고, 벡터, 패턴
    └── icons/      ← 아이콘 SVG
```

### 5-2. 파일명 규칙

| 유형 | 규칙 | 예시 |
|------|------|------|
| 이미지 | [섹션]-[용도].jpg | hero-bg.jpg, menu-grilled-01.jpg |
| 로고 | logo-[방향]-[색상].svg | logo-horizontal-white.svg |
| 아이콘 | icon-[이름].svg | icon-instagram.svg, icon-phone.svg |
| 패턴 | pattern-[이름].png | pattern-traditional.png |

### 5-3. MCP 에셋 추출 지시

- Figma MCP(get_design_context)에서 에셋 다운로드 URL이 제공되면 assets/ 폴더에 저장해 주세요.
- MCP에서 추출할 수 없는 에셋은 플레이스홀더로 대체하고, `<!-- TODO: [파일명] -->` 주석을 남겨 주세요.
- 이미지 경로는 최종 파일명으로 미리 설정해 주세요. (파일만 교체하면 바로 적용되도록)
- 플레이스홀더 사용 시 CSS fallback 색상을 지정해 주세요. (예: `background: #888 url('assets/images/hero-bg.jpg') center/cover;`)

## 7. 시안과 다르게 적용할 사항

> 시안에는 이렇게 되어 있지만, 의도적으로 다르게 구현해야 하는 항목입니다.

| 항목 | 시안 | 실제 적용 | 이유 |
|------|------|----------|------|
| 모든 섹션 키카피 텍스트 | "100% 고객 만족 후기" (동일하게 표시) | 섹션 3의 텍스트 목록에 따라 각각 다르게 적용 | 텍스트 오버라이드 미적용 |
| 배경 이미지 섹션 키카피 색상 | #111 (컴포넌트 기본값) | white로 변경 | 어두운 배경 위이므로 흰색이어야 가독성 확보 |
| 배경 이미지 섹션 본문 색상 | #505050 (컴포넌트 기본값) | white로 변경 | 어두운 배경 위이므로 흰색이어야 가독성 확보 |

## 8. 디자이너 미확인 사항 처리 기준

> 시안에서 명확하지 않은 항목에 대해, AI가 어떻게 처리할지 사전에 결정합니다.

| 항목 | 처리 기준 |
|------|---------|
| 하단 문의 바 sticky 여부 | sticky 없이 구현 (메인 비주얼 하단 + 푸터 하단에만 표시) |
| GNB 스크롤 시 배경 | 흰색 배경 + box-shadow로 구현 |
| 어두운 배경 위 텍스트 색상 | 모두 white로 구현 (배경 이미지 섹션, 중간 배너 등) |
| 타겟 브라우저 | 모던 브라우저만 (IE 미지원) |
| 문의 폼 제출 시 동작 | alert("문의가 접수되었습니다.") 처리 (백엔드 미연동) |
| 창업절차 카드 테두리 | dashed (점선)으로 구현 |
| 곁들임 탭 메뉴 내용 | 구이류 탭과 동일한 카드 레이아웃, 메뉴명만 placeholder로 대체 |
| 고객 후기 리뷰 텍스트 | 시안에 표시된 텍스트 그대로 사용, 없으면 placeholder |
