# 업무요청서 작성 업무가이드

> AI 퍼블리싱을 위한 업무요청서를 Claude에게 맡겨서 작성하는 방법입니다.
> PM이 준비해야 할 것, Claude에게 넘기는 방법, 결과물 검수 방법을 안내합니다.

---

## 전체 흐름 (3단계)

```
[1단계] PM 준비 (10~15분)
    ↓
[2단계] Claude에게 프롬프트 전달 → 자동 작성 (5~10분)
    ↓
[3단계] PM 검수 및 수정 (10~15분)
    ↓
완성된 업무요청서 → 퍼블리싱 세션에서 사용
```

**예상 총 소요시간: 25~40분**

---

## 1단계: PM 준비 (10~15분)

### 1-1. Figma에서 확인할 것

| 확인 항목 | 확인 방법 | 예시 |
|----------|----------|------|
| Figma URL | 브라우저 주소창 복사 | `https://www.figma.com/design/Rsm420r5w1Q8ORRn0uFlIW` |
| PC 프레임 노드 ID | Figma에서 PC 프레임 선택 → URL의 `node-id=` 값 | `3:924` |
| Mobile 프레임 노드 ID | 같은 방법 | `33:791` |
| 모바일 GNB 열림 노드 ID | 같은 방법 | `3:167` |
| PC 프레임 사이즈 | 우측 패널에서 확인 | `1560 x 8775px` |

**노드 ID 확인 방법:**
1. Figma에서 해당 프레임을 클릭
2. 브라우저 URL에 `?node-id=3-924` 형태로 표시됨
3. `-`를 `:`로 변환하여 기록 (예: `3-924` → `3:924`)

### 1-2. 디자이너에게 확인할 것

| 확인 항목 | 질문 | 왜 필요한가 |
|----------|------|-----------|
| 텍스트 오버라이드 | "시안의 텍스트가 실제 내용이 맞나요?" | 컴포넌트 인스턴스의 텍스트가 미적용되는 경우가 많음 |
| 인터랙션 동작 | "이 버튼/탭/슬라이드는 어떻게 동작하나요?" | 정적 시안에서 알 수 없음 |
| 의도적 변경 사항 | "시안과 다르게 적용할 부분이 있나요?" | 시안 수정 전 선반영 필요 |
| 미확정 사항 | "아직 결정 안 된 부분이 있나요?" | AI가 임의 판단하지 않도록 |

**팁:** 디자이너 확인이 바로 안 되면, PM이 알고 있는 범위에서 먼저 작성하고 나중에 보완해도 됩니다.

### 1-3. PM이 직접 정리할 것

아래 내용을 메모장이나 텍스트 파일에 정리해 둡니다.

**필수:**
- [ ] Figma URL
- [ ] 노드 ID (PC, Mobile, 모바일 GNB 열림)
- [ ] 텍스트 오버라이드 상태 (정상 / 미적용)
- [ ] 미적용 시: 섹션별 실제 텍스트
- [ ] 인터랙션 목록 (필수 / 선택)

**선택 (있으면 적고, 없으면 비워두기):**
- [ ] 시안과 다르게 적용할 사항
- [ ] 디자이너 미확인 사항 처리 기준
- [ ] 추가 폰트 (Pretendard 외)
- [ ] 브레이크포인트 (768px 외 추가 필요 시)

## 2단계: Claude에게 프롬프트 전달 (5~10분)

### 2-1. 새 세션 열기

Claude Code에서 새 세션을 엽니다.

### 2-2. 프롬프트 작성

`업무요청서_작성_프롬프트.md` 파일의 프롬프트 템플릿을 복사합니다.

**`[  ]` 안의 값만 실제 값으로 바꿉니다:**

| 바꿀 항목 | 어디서 가져오나 |
|----------|--------------|
| 프로젝트명 | PM이 결정 |
| Figma URL | 1단계에서 복사 |
| 반응형 너비 | Figma PC 프레임 너비 |
| 노드 ID들 | 1단계에서 확인 |
| 실제 텍스트 | 1단계에서 정리 |
| 인터랙션 | 1단계에서 정리 |
| 저장 경로 | PM이 결정 |
| 양식 파일 경로 | `업무요청서_양식_v2.md`의 절대 경로 |

### 2-3. 전달 및 대기

- 프롬프트를 Claude에게 전달
- Claude가 Figma MCP로 시안을 분석하고 양식을 채움
- 약 5~10분 소요 (MCP 호출 횟수에 따라 다름)

### 2-4. 크레딧 참고

- Figma MCP `get_design_context` 호출: 섹션당 약 1회
- 12개 섹션 기준 약 12~15회 호출 예상
- Professional 플랜 기준 충분히 커버 가능

## 3단계: PM 검수 및 수정 (10~15분)

Claude가 작성한 업무요청서를 열어서 아래 항목을 확인합니다.

### 3-1. 필수 검수 체크리스트

| No | 검수 항목 | 확인 방법 | 중요도 |
|----|----------|----------|--------|
| 1 | **섹션 누락 없는지** | 섹션 목록(섹션 2)의 번호가 시안과 일치하는지 | 최우선 |
| 2 | **텍스트가 맞는지** | 섹션별 텍스트(섹션 3)가 실제 내용과 일치하는지 | 최우선 |
| 3 | **인터랙션 빠짐없는지** | 필수 인터랙션(섹션 4-1)이 의도한 대로 적혀 있는지 | 필수 |
| 4 | **충돌 우선순위** | 섹션 6-3의 우선순위 테이블이 맞는지 | 필수 |
| 5 | **시안 변경 사항** | 섹션 7의 내용이 의도와 일치하는지 | 해당 시 |
| 6 | **미확인 사항 처리** | 섹션 8의 처리 기준이 합리적인지 | 권장 |

### 3-2. 흔한 수정 사항

| 상황 | 대처 |
|------|------|
| 섹션이 누락됨 | 섹션 2 목록에 추가 + 섹션 3에 텍스트 추가 |
| 텍스트가 시안 값 그대로 들어감 | PM이 제공한 실제 텍스트로 교체 |
| 인터랙션이 너무 적음 | 필요한 동작을 섹션 4에 추가 |
| MCP가 읽을 수 있는 px 값이 들어감 | 해당 수치 삭제 (색상, 크기, 간격 등) |
| 장식 섹션(배너, 패턴 등)이 빠짐 | 섹션 2에 독립 섹션으로 추가 |

### 3-3. 수정 방법

- 직접 .md 파일을 편집해도 되고
- Claude에게 "섹션 2에 중간 배너를 추가해줘" 같이 요청해도 됩니다

## 완성 후: 퍼블리싱 세션에서 사용

업무요청서가 완성되면, 별도의 새 세션에서 퍼블리싱을 진행합니다.

### 퍼블리싱 프롬프트 예시

```
[브랜드명] 랜딩페이지를 퍼블리싱해줘.

## 작업 정보
- 작업 위치: /프로젝트/경로/test-c/
- 결과물: index.html + style.css + script.js
- 에셋 폴더: /프로젝트/경로/test-c/assets/ (images, svg, icons)

## 참조 문서 (반드시 읽고 구현할 것)
- /프로젝트/경로/업무요청서.md

## Figma MCP 활용 (Professional 플랜, 크레딧 사용 가능)
- get_design_context로 각 섹션의 코드와 스타일 정보를 추출해줘
- get_variable_defs로 디자인 토큰을 가져와서 CSS 변수로 활용
- MCP에서 에셋 다운로드 URL이 있으면 assets/ 폴더에 저장
- 크레딧 충분하니 섹션마다 꼼꼼하게 호출해도 됨

## 핵심 요구사항
- 순수 HTML + CSS (프레임워크 없음)
- 사람이 수정/유지보수할 수 있는 코드
- 업무요청서의 전체 섹션을 빠짐없이 구현
- 업무요청서의 텍스트와 시안이 다를 경우 업무요청서 기준

전체 페이지를 완성해줘.
```

## 파일 구성 요약

```
프로젝트/
├── 업무요청서_양식_v2.md           ← 빈 양식 (재사용)
├── 업무요청서_작성_프롬프트.md      ← 양식 채우기용 프롬프트 (재사용)
├── 업무요청서_작성_업무가이드.md    ← 이 파일 (참조용)
└── [프로젝트명]_업무요청서v2.md    ← 완성된 업무요청서 (프로젝트별 생성)
```

| 파일 | 누가 사용 | 언제 사용 | 재사용 |
|------|----------|----------|--------|
| 양식 | Claude | 업무요청서 작성 시 | 모든 프로젝트 |
| 프롬프트 | PM → Claude | 새 세션에서 업무요청서 작성 요청 시 | 모든 프로젝트 |
| 업무가이드 | PM | 처음 프로세스 익힐 때 | 참조용 |
| 완성 업무요청서 | Claude | 퍼블리싱 세션에서 참조 | 해당 프로젝트만 |

---

## 주의사항

1. **업무요청서 작성 세션 ≠ 퍼블리싱 세션**: 반드시 별도 세션에서 진행
2. **양식에 px 값 쓰지 않기**: MCP가 시안에서 직접 읽으므로 중복 기재하면 충돌 발생
3. **장식 섹션 빠뜨리지 않기**: 배경 이미지, 중간 배너, 패턴 구분선 등도 독립 섹션으로 포함
4. **텍스트 오버라이드 반드시 확인**: 이전 테스트에서 가장 큰 문제였음
