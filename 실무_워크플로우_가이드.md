## 2. AI가 잘 하는 것 vs 사람이 해야 하는 것

### 2-1. AI가 잘 하는 것 (4개 테스트 공통 확인)

**HTML 구조 생성**
- 시맨틱 태그 사용 (`<header>`, `<nav>`, `<section>`, `<footer>`)
- 섹션별 명확한 id/class 부여 (`id="brand"`, `id="difference"`, `id="contact"`)
- BEM 네이밍 일관 적용 (`.gnb__link`, `.menu-card__img`, `.contact-form__label`)
- 섹션 구분 주석 자동 생성
- 모바일 GNB 오버레이 구조 (열기/닫기 + ARIA 속성)

**CSS 스타일링**
- CSS 변수 시스템 자동 정의 (디자인 토큰 → `:root` 변수)
- 기본 반응형 768px 브레이크포인트 처리
- 글래스모피즘, 카드 그리드, 테이블 등 정적 컴포넌트 완성
- Pretendard 웹폰트 CDN 연동
- 패턴 배경, 오버레이, 그라데이션 등 장식 요소

**기본 JavaScript**
- GNB 스크롤 시 배경색 변경 (`.scrolled` 또는 `.gnb--transparent`)
- 모바일 햄버거 메뉴 열기/닫기 + body overflow 제어
- 탭 UI 활성 상태 토글
- 스무스 스크롤 (네비게이션 앵커 이동)

### 2-2. AI가 못 하는 것 (공통 미구현) — test-4에서 일부 개선

| 영역 | 구체적 항목 | 사람이 해야 하는 작업 | test-4 개선 |
|------|-----------|---------------------|:-----------:|
| **슬라이더** | Swiper.js 연동 | CDN 추가 → swiper 구조 적용 → JS 초기화 | **커스텀 구현** (Swiper 교체 권장) |

| **폼 서버 연동** | reCAPTCHA + AJAX 전송 | 서버 API + 캡챠 키 설정 | 미개선 |

### STEP 3. AI 산출물 검수

**실행자**: 사람 (퍼블리셔)
**소요 시간**: 30분
**목적**: AI 결과물과 완성본 사이의 차이를 빠짐없이 파악

#### 검수 체크리스트

**A. 섹션 완성도 (가장 중요)**

```
□ 전체 섹션 수가 시안과 일치하는가
  → 테스트 결과: AI는 평균 1~2개 섹션을 누락하거나 단순화함
  → 가장 빈번한 누락: 배경이미지 섹션 (블러+누끼+텍스트 구성)
  → 확인 방법: 시안을 위에서 아래로 스크롤하며 AI 결과물과 대조

□ 각 섹션 내부 요소가 시안과 일치하는가
  → 카드 개수, 탭 개수, 테이블 행 수 등
  → 텍스트 내용이 시안과 같은가 (AI가 임의로 바꾸는 경우 있음)
```

**B. 이미지/에셋 상태**

```
□ 이미지가 실제로 표시되는가 (alt 텍스트만 보이지 않는지)
  → MCP가 추출에 실패한 이미지가 있을 수 있음
  → 깨진 이미지 위치를 기록

□ 이미지 크기/비율이 시안과 맞는가
  → 특히 히어로 배경, 음식 사진 카드, 중간 배너

□ SVG 로고가 정상 표시되는가
  → MCP 추출 시 SVG가 깨지는 경우가 간혹 있음

□ 플레이스홀더로 남은 이미지가 있는가
  → 추출 실패한 에셋 목록 작성
```

**C. 인터랙션 동작**

```
□ GNB 메뉴 클릭 시 해당 섹션으로 스크롤 이동하는가
□ 스크롤 시 GNB 배경색이 변경되는가
□ 모바일 햄버거 메뉴가 열리고 닫히는가
□ 탭 클릭 시 활성 상태가 바뀌는가
  → 주의: 탭 "활성 스타일"만 바뀌고 실제 콘텐츠는 안 바뀔 수 있음
□ 슬라이더 화살표가 존재하는가 (동작은 안 해도 UI는 있어야 함)
```

**D. 모바일 확인**

```
□ DevTools에서 375px로 설정 후 전체 스크롤
□ 요소가 화면 밖으로 넘치지 않는가
□ 텍스트가 잘리거나 겹치지 않는가
□ 모바일 GNB가 50px 높이인가
  → 테스트 결과: AI가 64px로 설정하는 경우 있음 (PC와 동일하게)
```

**E. 색상/타이포**

```

  → CSS 변수가 시안의 디자인 토큰과 일치하는가
□ 제목 폰트 크기가 적절한가
  → AI 기본: Pretendard만 적용. 시안에 다른 폰트가 있으면 기록
□ 본문 폰트 크기 확인 (시안 17px vs AI 16~20px 편차 가능)
```

### STEP 4. 누락 섹션·구조 보강

**실행자**: AI + 사람
**소요 시간**: 1~2시간

테스트 결과 AI는 평균 1~2개 섹션을 누락하거나 단순화합니다.
가장 빈번한 누락 패턴:

| 누락 패턴 | 발생 빈도 | 원인 추정 |
|-----------|:--------:|----------|
| 배경이미지 섹션 전체 누락 | 2/3 테스트 | 독립 콘텐츠가 아닌 장식 섹션이라 AI가 건너뜀 |
| 탭 콘텐츠 교체 미구현 | 2/3 테스트 | 시각적 토글만 구현, 실제 DOM 교체 누락 |
| 폰 목업 구조 누락 | 2/3 테스트 | 복잡한 z-index 레이어 구성을 단순화 |
| 수평 dashed 연결선 | 3/3 테스트 | CSS로 구현하기 까다로운 요소 |

#### 방법 1: AI에게 추가 요청 (권장)

같은 Claude Code 세션에서 이어서 요청합니다.

```
   - [구성 요소 설명]

2. [섹션명]의 탭을 클릭하면 실제 콘텐츠가 교체되어야 해.
   현재는 탭 스타일만 바뀌고 내용은 그대로야.

3. 모바일 GNB 높이를 64px에서 50px로 수정해줘.
```

### STEP 5. 인터랙션 추가

**실행자**: 사람 (퍼블리셔)
**소요 시간**: 3~4시간

테스트 결과, 아래 인터랙션은 AI가 한 번도 구현하지 못했습니다.
사람이 직접 추가해야 합니다.

#### 5-1. Swiper 슬라이더 (필수 — 가장 빈번한 보강)

**상황**: AI는 카드를 그리드로 나열만 합니다. 좌우 슬라이드는 동작하지 않습니다.

```html
<!-- ① head에 CSS 추가 -->
<link rel="stylesheet"
  href="https://cdn.jsdelivr.net/npm/swiper@11/swiper-bundle.min.css">

<!-- ② 기존 카드 그리드를 swiper 구조로 교체 -->
<!-- 변경 전 (AI 생성 코드) -->
<div class="menu-section__cards">
  <div class="menu-card">...</div>
  <div class="menu-card">...</div>
  <div class="menu-card">...</div>
  <div class="menu-card">...</div>
</div>

<!-- 변경 후 -->
<div class="swiper menu-swiper">
  <div class="swiper-wrapper">
    <div class="swiper-slide"><div class="menu-card">...</div></div>
    <div class="swiper-slide"><div class="menu-card">...</div></div>
    <div class="swiper-slide"><div class="menu-card">...</div></div>
    <div class="swiper-slide"><div class="menu-card">...</div></div>
  </div>
  <div class="swiper-button-prev"></div>
  <div class="swiper-button-next"></div>
</div>

<!-- ③ body 하단에 JS 추가 -->
<script src="https://cdn.jsdelivr.net/npm/swiper@11/swiper-bundle.min.js"></script>
<script>
new Swiper('.menu-swiper', {
  slidesPerView: 4,
  spaceBetween: 20,
  navigation: {
    nextEl: '.swiper-button-next',
    prevEl: '.swiper-button-prev'
  },
  breakpoints: {
    0:   { slidesPerView: 1.2 },
    768: { slidesPerView: 2 },
    1024:{ slidesPerView: 4 }
  }
});
</script>
```

**주의사항**:
- AI가 만든 `.menu-card`의 width를 고정값에서 `auto` 또는 `100%`로 변경해야 함
- 기존 grid/flex 레이아웃 CSS를 제거하거나 swiper에 맞게 수정

#### 5-2. AOS 스크롤 진입 효과 (효과 대비 작업량 최소)

```html
<!-- ① head에 CSS 추가 -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/aos@2.3.4/dist/aos.css">

<!-- ② 각 섹션에 data-aos 속성 추가 -->
<section class="brand" id="brand" data-aos="fade-up">
<section class="difference" id="difference" data-aos="fade-up" data-aos-delay="100">
<div class="menu-card" data-aos="fade-up" data-aos-delay="100">

<!-- ③ body 하단에 JS 추가 -->
<script src="https://cdn.jsdelivr.net/npm/aos@2.3.4/dist/aos.js"></script>
<script>
AOS.init({
  duration: 800,
  easing: 'ease-out-cubic',
  once: true
});
</script>
```

**효과**: `data-aos` 속성 하나로 섹션이 스크롤 시 서서히 나타나는 효과.
레퍼런스와의 시각적 차이를 빠르게 좁힐 수 있는 가성비 최고 작업입니다.

#### 5-3. GSAP ScrollTrigger (고급 — 필요한 경우만)

```html
<!-- 고객후기 섹션의 스크롤 기반 슬라이드 등 고급 효과 -->

<script src="https://cdn.jsdelivr.net/npm/gsap@3.12/dist/ScrollTrigger.min.js"></script>

    start: 'top top',

  }

</script>
```

#### 5-5. 탭 콘텐츠 실제 교체

AI는 탭의 "활성 스타일"만 토글하고, 실제 콘텐츠 교체는 구현하지 않는 경우가 많습니다.

```javascript
// AI가 만든 코드 (스타일만 토글)
tabBtns.forEach(btn => {
  btn.addEventListener('click', () => {
    tabBtns.forEach(b => b.classList.remove('active'));
    btn.classList.add('active');
  });
});

// 사람이 추가할 코드 (콘텐츠 교체)
tabBtns.forEach(btn => {
  btn.addEventListener('click', () => {
    const target = btn.dataset.tab;  // "grilled" or "sides"
    tabBtns.forEach(b => b.classList.remove('active'));
    btn.classList.add('active');
    // 콘텐츠 교체
    tabPanels.forEach(panel => {
      panel.style.display = panel.id === target ? 'block' : 'none';
    });
    // Swiper가 있다면 재초기화
    if (window.menuSwiper) window.menuSwiper.update();
  });
});
```

```css
  /* ... 전 섹션 모바일 대응 ... */
}
```

#### 사람이 추가하는 중간 단계

```css
/* ① 소형 데스크탑 — 사이드 여백 축소 */
@media (max-width: 1280px) {
  :root {
    --section-px: 60px;  /* 130px → 60px */
  }
  .hero__title { font-size: 46px; }  /* 52→46 */
}

/* ② 태블릿 가로 — 그리드 열 수 조정 */
@media (max-width: 1024px) {
  .diff__cards { grid-template-columns: repeat(2, 1fr); } /* 5→2열 */
  .process__grid { grid-template-columns: repeat(3, 1fr); } /* 5→3열 */
  .hero__title { font-size: 40px; }
}

/* ③ AI가 만든 모바일 768px (이미 존재) */
@media (max-width: 768px) {
  /* AI 생성 코드 유지 */
}

/* ④ 소형 모바일 */
@media (max-width: 480px) {
  .hero__title { font-size: 28px; }
  .brand__title { font-size: 24px; }
  .process__grid { grid-template-columns: 1fr; } /* 1열 */
  .cost-table { font-size: 13px; }
}
```

1. 브라우저 DevTools에서 화면 너비를 1280 → 1024 → 768 → 480으로 줄이며 확인
2. 레이아웃이 어색한 지점에서 미디어쿼리 추가
3. 타이포 크기를 단계적으로 축소
4. 그리드 열 수를 해상도에 따라 조정

## 8. 흔한 문제와 해결법

### 8-1. AI가 섹션을 누락했을 때

**원인**: AI가 장식적 섹션(배경이미지 전환, 중간 배너 등)을 "독립 콘텐츠가 아님"으로 판단하고 생략

**해결**: 프롬프트에 전체 섹션 목록을 명시
```
전체 12개 섹션을 빠짐없이 구현해줘:
GNB, 메인비주얼, 브랜드소개, 배경이미지, 차별점,
상차림, 고객후기, 중간배너, 창업절차, 창업비용,
문의하기, 푸터
```

### 8-2. MCP 에셋 추출이 실패했을 때

**원인**: Figma 파일의 에셋이 컴포넌트화되지 않았거나, MCP 로컬 서버 연결 문제

**해결**:
1. Figma 데스크탑 앱에서 해당 파일이 열려 있는지 확인
2. Dev Mode가 활성화되어 있는지 확인
3. 실패한 에셋만 수동으로 Figma에서 Export

### 8-3. 모바일 GNB 높이가 맞지 않을 때

**원인**: AI가 PC와 모바일 GNB 높이를 구분하지 못함 (test-3에서 64px 유지)

**해결**: 미디어쿼리에서 직접 수정
```css
@media (max-width: 768px) {
  .gnb { height: 50px; }
  .gnb__logo-img { height: 30px; }
}
```

### 8-4. 탭이 스타일만 바뀌고 콘텐츠가 안 바뀔 때

**원인**: AI는 탭 버튼의 활성 클래스만 토글하고, 연결된 콘텐츠 패널 표시/숨김을 구현하지 않음 (2/3 테스트에서 발생)

**해결**: STEP 5-5 탭 콘텐츠 실제 교체 코드 참조

### 8-5. 키카피(제목) 텍스트가 시안과 다를 때

**원인**: Figma MCP가 텍스트 오버라이드를 정확히 읽지 못하는 경우가 있음

**해결**: 시안과 대조하여 수동으로 텍스트 교체. 또는 프롬프트에 핵심 텍스트를 명시:
```
- 메인비주얼 제목: "돼지 갈매기살의 새로운 기준"
- 브랜드 제목: "갈매기대도를 소개합니다"
```

### 8-6. 추출된 이미지 용량이 너무 클 때

**원인**: MCP가 원본 해상도로 추출 (test-3에서 hero-bg.png 5.1MB, diff-store.png 5.1MB)

**해결**: 이미지 최적화 도구로 압축
```bash
# ImageMagick 사용 예시
convert assets/images/hero-bg.png -quality 80 -resize 1920x assets/images/hero-bg.jpg
# 또는 squoosh.app, tinypng.com 등 온라인 도구
```

```
프로젝트/
```

- **양식 v2**: MCP가 읽을 수 있는 수치(색상, 크기, 간격)를 제거. 시안에서 알 수 없는 정보만 기재.

- **업무가이드**: PM 준비(10~15분) → Claude 자동 작성(5~10분) → PM 검수(10~15분) 3단계 절차.
