### 2. 에셋 (이미지/비디오)

- 실사 음식 사진 (구이류 6종, 곁들임 9종)
- 매장 외관/내부 사진 4장+
- 비디오 배경 1개 (메인비주얼)
- SVG 로고 (가로형/세로형/파비콘)
- 아이콘 SVG 다수
- 붓터치 SVG, 패턴 이미지

### 3. 인터랙션

| 기능 | 구현 방식 |
|------|----------|
| 메뉴 스크롤 이동 | jQuery animate scrollTop |
| 스크롤 시 GNB 배경 변경 | onscroll gnb-white 클래스 토글 |
| 모바일 햄버거 메뉴 | jQuery fadeIn/fadeOut |
| 상차림 탭 전환 | 구이류/곁들임 실제 컨텐츠 교체 + 별도 Swiper 인스턴스 |
| 슬라이더 | **Swiper.js 11** (좌우 드래그 + 화살표) |
| 스크롤 애니메이션 | **GSAP 3.12 ScrollTrigger** + **AOS 2.3.1** |
| 텍스트 효과 | SplitType + CSS wave 키프레임 |
| 폼 제출 | reCAPTCHA v3 + AJAX 서버 전송 |

## 결론

reference는 실무 퍼블리셔가 제작한 **완전한 제품**입니다.
AI 테스트 결과물들은 **정적 구조(HTML/CSS)**는 잘 재현하지만,
**동적 인터랙션**(슬라이더, 스크롤 애니메이션)과 **실사 에셋**(사진, 비디오)은 사람의 보강이 필요합니다.

그중 test-3 (Figma 로컬 MCP)만 유일하게 **에셋을 자동 추출**하여,
사용자 눈에 보이는 화면이 레퍼런스와 가장 유사합니다.
