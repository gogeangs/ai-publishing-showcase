# 퍼블리싱 테스트 프롬프트 문안

## 실행 방법

> **각 테스트는 별도의 새 Claude Code 창에서 실행합니다.**
> - 이전 대화 컨텍스트가 없으므로 각 프롬프트는 완전히 독립적/자기완결적으로 작성됨
> - 새 Claude Code 창을 열고 → 해당 프롬프트를 복사-붙여넣기 → 실행
> - test-3은 2단계이므로 **같은 창**에서 A단계 완료 후 B단계 프롬프트를 이어서 입력

## 실행 순서

1. 새 창 → test-1 프롬프트 붙여넣기 → 완료 대기
2. 새 창 → test-2 프롬프트 붙여넣기 → 완료 대기
3. Figma 데스크탑 앱에서 시안 열기 + MCP 로컬 서버 활성화 확인 → 새 창 → test-3 프롬프트 붙여넣기 → 완료 대기
4. 이 창(PM 세션)으로 돌아와서 채점 및 비교

## 전체 공통 조건

> **모든 테스트에 아래 조건이 적용됩니다.**
> - 사람이 직접 하드코딩하거나 유지보수할 수 있도록 **수정하기 쉬운 코드**로 작성
> - HTML 구조가 직관적이고, CSS 클래스명이 명확하며, 섹션 구분이 잘 되어야 함
> - 매직넘버 대신 CSS 변수 사용, 반복 패턴은 공통 클래스로 추출
> - 주석으로 각 섹션/컴포넌트 구분을 명확히 표기

## test-1: .md 업무요청서 기반 + Figma MCP 보조

### 프롬프트 (복사용)

```

- 작업 위치: /home/go861/ITSO/test-1/
- 결과물: index.html + style.css (+ 필요 시 script.js)
- 에셋 폴더: /home/go861/ITSO/test-1/assets/ (images, svg, icons 하위 폴더 이미 생성됨)

## 핵심 요구사항
- 순수 HTML + CSS 퍼블리싱 (프레임워크 없음)
- 반응형: PC(1560px) + Mobile(375px) 2단계, breakpoint 768px
- 이미지 에셋은 아직 없으므로 플레이스홀더 사용 (업무요청서 10장의 플레이스홀더 운용 방침 참조)
- 업무요청서의 구현 권장 순서(7장)에 따라 Phase 1~3 순서로 구현
- **사람이 직접 수정/유지보수할 수 있도록 코드를 작성할 것:**
  - HTML 구조를 직관적으로 — 섹션별 시맨틱 태그, 명확한 class명
  - CSS는 매직넘버 대신 변수 사용, 반복 패턴은 공통 클래스로 추출
  - HTML/CSS 양쪽에 섹션 구분 주석을 달아 어디가 어딘지 바로 알 수 있게
  - 코드를 읽는 사람이 Figma 시안 없이도 구조를 이해할 수 있는 수준
```

```
- 이미지 출처가 로컬 서버로 설정되어 있으니, MCP에서 제공하는 에셋 다운로드 URL이 있으면 assets/ 폴더에 저장해줘
- 섹션별로 get_design_context를 호출해서 정확한 수치와 스타일을 확인하면서 구현해줘
- MCP 토큰/크레딧은 충분하니 섹션마다 꼼꼼하게 호출해도 됨

## 요구사항

- 반응형: PC + Mobile 2단계 (breakpoint 768px)

- **사람이 직접 수정/유지보수할 수 있도록 코드를 작성할 것:**
  - HTML 구조를 직관적으로 — 섹션별 시맨틱 태그, 명확한 class명
  - CSS는 매직넘버 대신 변수 사용, 반복 패턴은 공통 클래스로 추출
  - HTML/CSS 양쪽에 섹션 구분 주석을 달아 어디가 어딘지 바로 알 수 있게
  - 코드를 읽는 사람이 Figma 시안 없이도 구조를 이해할 수 있는 수준
```
